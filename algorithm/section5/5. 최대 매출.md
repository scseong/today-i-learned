## 최대 매출

### 1. 문제 정보

- **문제 제목**: 4. 최대 매출
- **카테고리**: 효율성, 슬라이딩 윈도우

### 2. 문제 설명

- **핵심 목표**: N일간의 매출 기록에서 연속된 K일 동안의 매출 합 중 최댓값을 찾는다.
- **제한 사항**:
  - 매출 기록 수 N (최대 100,000)
  - 연속 일수 K (최대 N)
- **입출력 예시**:
  - Input: K = 3, [12 15 11 20 25 10 20 19 13 15]
  - Output: 56

### 3. 접근 방식 및 알고리즘

- **사용한 알고리즘**:
  1. 슬라이딩 윈도우
- **풀이 로직**:
  1. 반복문을 사용하여 배열의 0번부터 $K-1$번 인덱스까지의 합을 먼저 구한다.
  2. i를 K부터 끝까지 순회하며 한 칸씩 오른쪽으로 이동한다.
  3. 이전 합계에 새로 들어온 arr[i]는 더하고 나가는 arr[i-k]는 뺀다
  4. Math.max를 통해 기존 최댓값과 현재 합계를 비교

### 4. 성능 측정 결과 (Performance)

| **구분**       | **실행 시간 (평균)** | **시간 복잡도** | **공간 복잡도** |
| -------------- | -------------------- | --------------- | --------------- |
| **Solution 1** | **0.055ms**          | O(N)            | O(1)            |

## 💻 JavaScript 풀이 코드

Solution 1: 초기 k일의 합을 구한 뒤 새로운 값을 추가하고 이전 값 제거하여 최댓값을 갱신

```js
function solution1(k, arr) {
  let answer = 0;
  let count = k;
  let sum = 0;

  // 1. 초기 윈도우(0 ~ k-1)의 합 구하기
  while (count--) sum += arr[count];
  answer = sum;

  // 2. 슬라이딩 윈도우: k번 인덱스부터 끝까지 전진
  for (let i = k; i < arr.length; i++) {
    // 3. 윈도우 갱신: 새로운 원소 합산 - 범위 벗어난 원소 차감
    sum = sum + arr[i] - arr[i - k];

    // 4. 최댓값 업데이트
    answer = Math.max(sum, answer);
  }

  return answer;
}
```
